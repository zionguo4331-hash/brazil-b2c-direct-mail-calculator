# 小白用户使用说明

## 你可以怎么用这套工具

这套工具现在的主系统就是 GitHub 前端本身。

也就是说，不使用 Dify，你也可以完成：

- 新增报价卡
- 上传 Excel
- 生成 AI 可读文本
- 用本地规则识别生成报价草稿
- 人工确认报价卡
- 保存到报价库
- 单 SKU 测算
- 批量 SKU 测算
- 多报价对比

建议按下面顺序使用：

1. 先新增报价卡，或导入 AI 输出的 `quote_cards.json`
2. 确认报价卡状态
3. 做单 SKU 测算
4. 再做批量 SKU 测算
5. 最后做多报价对比
6. 导出报价库和结果做备份

## AI 报价解析只做一次，而且是可选增强

AI / Dify 只负责在“新增报价”或“更新报价”时解析一次原始报价。

确认后的报价卡会保存到本地报价库里，后续：

- 单 SKU 测算
- 批量 SKU 测算
- 多报价对比

都会直接复用这份报价库，不需要每次重新解析原始报价。

如果本地规则识别结果已经足够清楚，你完全可以不使用 Dify。

## 如何新增报价卡

1. 进入“报价卡管理”
2. 点击“新增报价卡”
3. 填写报价名称、货代、模式、计费方式和主要价格
4. 如果关键字段齐全，可以点击“确认报价卡”
5. 如果字段还不完整，系统会自动标记为“需复核”

## 如何导入 AI 输出的 quote_cards.json

1. 进入“报价卡管理”
2. 找到“导入 AI 解析结果”
3. 上传 JSON 文件或直接粘贴 JSON 文本
4. 点击“预览 AI 导入结果”
5. 先看：
   - 成功识别了几张报价卡
   - 哪些字段高置信
   - 哪些字段低置信
   - 哪些字段需要人工确认
6. 点击“保存到报价库”

注意：

- AI 导入的报价卡默认是 `needs_review`
- 不会自动变成 `confirmed`

## 如何上传 Excel 并生成报价草稿

1. 进入“报价库 Quote Library”
2. 找到“Excel 报价预处理”
3. 上传 `.xlsx`
4. 勾选要处理的 Sheet
5. 系统会同时做两件事：
   - 生成 AI 可读文本
   - 运行本地规则识别，生成 Quote Card 草稿
6. 草稿默认 `needs_review`
7. 你可以直接编辑和确认，不一定要走 Dify

## 如何使用本地规则识别结果

本地规则识别生成的报价卡来源会标记为 `rule_extracted`。

这类报价卡：

- 默认是 `needs_review`
- 可以保存到报价库
- 可以继续人工编辑
- 确认后才能正式参与测算

## 如何复制 AI 可读文本给 Dify

如果 Excel 很复杂，本地规则识别不够清楚，可以：

1. 在“Excel 报价预处理”中复制 AI 可读文本
2. 把这段文本发给 Dify 报价解析助手
3. 让 Dify 输出 `quote_cards.json`
4. 再把 JSON 导回本工具

这一步是可选增强，不是主流程必经步骤。

## 如何确认报价卡

只有关键字段完整、且未过期的报价卡，才建议作为正式定价依据。

你会在页面中看到：

- 缺失字段
- 风险字段
- 是否可以正式参与测算

如果报价过期，系统会自动提示你重新向货代确认。

## 如何确认 AI 解析的报价卡

1. 进入“报价卡管理”
2. 在“AI 报价卡确认”里选择一张 AI 报价卡
3. 检查关键价格字段
4. 查看 AI 置信度
5. 回答系统给出的确认问题
6. 填写确认人
7. 点击“确认报价卡”

只有在以下条件都满足时，报价卡才能变成 `confirmed`：

- 关键字段校验通过
- 必填确认问题没有“不确定”
- 报价未过期

## 为什么不要每次测算都重新解析报价

因为每次重新解析会带来三个问题：

1. 结果不稳定
2. 难以复盘同一张报价卡版本
3. 容易把试算草稿和正式报价混在一起

更稳的做法是：

- 先解析一次
- 再人工确认
- 然后把确认后的报价卡放进报价库复用

## 同事如何导入同一份报价库

1. 你导出报价库 JSON
2. 发给同事
3. 同事打开工具
4. 进入“数据导入导出”
5. 导入这份报价库 JSON
6. 就可以继续使用同一批报价卡

## 如何上传 SKU

1. 进入“批量 SKU 测算”
2. 点击上传 CSV
3. 先看系统预览是否正确
4. 再选择一个或多个报价卡
5. 点击“运行批量测算”

## 如何做批量测算

批量测算会按下面逻辑执行：

- 每个 SKU
- 配每一张已选报价卡
- 生成一条结果

结果里会看到：

- 运费
- CIF
- 税费
- 总成本
- 净利润
- 净利率
- 风险等级
- 是否需要人工复核

## 如何做多报价对比

1. 进入“多报价对比”
2. 选择“当前单 SKU”或“最近一次 CSV 预览”
3. 勾选多个报价卡
4. 点击“对比多个报价”

系统会自动标记：

- lowest_freight
- highest_profit
- best_margin
- needs_review
- expired_quote

请注意：最低运费不等于最佳方案，还要看税费、平台费、尾程费、净利润和报价状态。

## 如何看风险提示

常见提示包括：

- QUOTE_NEEDS_REVIEW：报价卡还没确认
- QUOTE_EXPIRED：报价已过期
- QUOTE_KEY_FIELD_MISSING：关键字段缺失
- CIF_NEAR_50_USD_THRESHOLD：CIF 接近 50 美元红线
- NEGATIVE_PROFIT：当前售价亏损
- BELOW_TARGET_MARGIN：净利率低于目标

页面会用中文解释这些风险，不需要死记代码。

## 如何导出结果

你可以导出：

- 报价库 JSON
- 单张报价卡 JSON
- 批量测算结果 CSV
- SKU CSV 模板
- 报价卡 CSV 模板

## 如何把报价库给同事

1. 在“数据导入导出”里导出全部报价库 JSON
2. 发给同事
3. 同事在自己的浏览器里导入这份报价库 JSON
4. 导入后就能继续使用同一批报价卡

## 术语解释

重点理解这些词就够用了：

- SKU：单个产品货号
- Quote Card：报价卡
- Quote Library：报价库
- CIF：商品成本 + 国际运费 + 保险
- II：进口关税
- ICMS：巴西州税
- 3PF：第三方履约
- CPA：广告获客成本
- Landed Cost：落地成本

## 重要提醒

当前数据保存在本机浏览器中：

- 换电脑可能丢失
- 清缓存可能丢失
- 建议定期导出报价卡 JSON 备份
