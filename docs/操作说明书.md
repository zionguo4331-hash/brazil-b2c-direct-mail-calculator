# 巴西 SKU 利润与物流报价决策器操作说明书

## 1. 这是什么工具

这是一套给跨境电商运营、采购、供应链和财务同学使用的前端测算工具。

它主要解决三件事：

1. 把复杂的货代报价整理成系统可复用的报价卡或报价包。
2. 用这些报价卡反复做单 SKU、批量 SKU 和多报价对比测算。
3. 帮团队在定价前快速判断哪种履约方案更稳、更赚、更适合正式报价。

这套工具是纯前端工具，数据保存在当前浏览器本机中。

## 2. 这套工具最适合谁

适合以下同学：

1. 跨境电商运营
2. 采购
3. 供应链
4. 财务测算
5. 需要做巴西物流报价比较的人

如果你不会看复杂物流报价表，也可以使用这套工具。系统会尽量自动识别价格结构，你只需要确认少量业务问题。

## 3. 整体使用流程

建议按下面顺序使用：

1. 先进入“报价卡管理”
2. 新增报价方式四选一：
   - 手动新增报价卡
   - 上传 Excel 报价文件生成草稿
   - 导入报价库 JSON
   - 导入 AI 解析结果 JSON
3. 把草稿保存到报价库
4. 进入“报价确认”，确认少量业务问题
5. 报价状态变成“已确认”
6. 再去做单 SKU 测算、批量 SKU 测算或多报价对比
7. 最后导出报价库 JSON 或导出测算结果 CSV

最重要的一条原则：

AI 或规则识别只在“新增报价 / 更新报价”时使用一次。

确认后的报价卡会保存到本地报价库，后续所有测算都直接复用它，不需要每次重新解析原始报价。

## 4. 主页面几个模块分别做什么

### 首页 / 使用指引

适合第一次打开工具时看一遍，快速了解整体流程。

### 单 SKU 测算

适合：

1. 测一个产品
2. 快速看利润
3. 验证某个报价卡是否合适

### 批量 SKU 测算

适合：

1. 一次测多个 SKU
2. 生成一批报价结果
3. 导出 CSV 给团队继续分析

### 报价卡管理

这是整个工具最重要的模块。

你可以在这里：

1. 新增报价卡
2. 上传 Excel 识别报价草稿
3. 导入 AI 解析结果
4. 做报价确认
5. 管理报价库

### 多报价对比

适合比较：

1. 同一个 SKU 在不同货代下的表现
2. 同一批 SKU 在不同报价方案下的表现

### 术语与公式说明

适合不熟悉 CIF、II、ICMS、3PF、CPA 等术语的同学查看。

### 数据导入导出

适合：

1. 备份报价库
2. 把报价库发给同事
3. 导入同事发来的报价库
4. 清空本地数据

## 5. 最推荐的用法

如果你是第一次用，建议按下面路径操作：

1. 进入“报价卡管理”
2. 上传货代 Excel 报价
3. 点击保存草稿并进入报价确认
4. 只确认业务问题，不要逐项核对所有价格字段
5. 把报价卡确认成“已确认”
6. 去“单 SKU 测算”选择这张报价卡试算
7. 如果没问题，再去“批量 SKU 测算”

## 6. 报价卡、报价包、报价库分别是什么意思

### Quote Card 报价卡

把货代报价整理成系统能计算的一张标准结构卡片。

### Quote Package 报价包

比普通报价卡更复杂，通常包含：

1. 主报价
2. 处理费
3. 尾程矩阵
4. 邮编区域表
5. 产品代码

PRC 复合报价一般更接近“报价包”的概念。

### Quote Library 报价库

就是你保存所有报价卡和报价包的地方。

后续单 SKU、批量 SKU、多报价对比都从报价库选。

## 7. 什么状态才可以正式拿来测算

报价一般有几种状态：

### 已确认

可以正式参与测算。

### 待确认

可以试算，但结果仅供参考，不建议直接用于正式定价。

### 已过期

可以参考旧价格，但不建议直接用于正式报价。

### 已禁用

不再参与默认选择。

## 8. 为什么系统不让你确认每个价格字段

因为小白用户不适合逐项判断：

1. 每公斤价格是否抄对
2. 尾程矩阵是否匹配
3. 重量段是否正确
4. 区域列是否完整
5. 邮编区间是否合理

这些应该优先由系统自动识别。

你真正需要确认的是业务问题，比如：

1. 这份报价是否适用于你的产品
2. 是否包含尾程
3. 是否包含清关
4. 是否包含进口税
5. 税费应该用哪种模式
6. 报价是否还在有效期内

## 9. 如何上传 Excel 报价文件

操作步骤：

1. 进入“报价卡管理”
2. 找到“Excel 报价预处理”
3. 上传 `.xlsx` 文件
4. 等系统读取完成
5. 系统会显示：
   - 识别到多少个 Sheet
   - 哪些 Sheet 有合并单元格
   - 是否识别到 PRC 主报价
   - 是否识别到尾程矩阵
   - 是否识别到邮编区域表
6. 查看“推荐下一步”
7. 点击“保存草稿并进入报价确认”

## 10. 上传 Excel 后应该怎么理解页面提示

### “Excel 已读取，识别到 X 个 Sheet”

表示文件已经成功读进浏览器了。

### “已识别 PRC 主报价、尾程矩阵和邮编区域表”

表示系统识别得比较完整，可以继续确认流程。

### “已识别 PRC 主报价，但尾程矩阵或邮编区域表缺失”

表示识别到了一部分，但不是完整报价包。仍然可以先保存草稿，再由物流或业务同事复核。

### “存在合并单元格，系统已尝试处理”

这不是失败。

它的意思是：

1. 你的 Excel 结构比较复杂
2. 系统已经尽量自动填充并识别
3. 后续不需要你逐项检查所有价格字段
4. 只需要重点确认少量业务问题

## 11. 什么叫“保存草稿并进入报价确认”

这个按钮做三件事：

1. 把规则识别出来的草稿写入报价库
2. 自动选中一张待确认草稿
3. 带你进入“报价确认”区域

所以：

上传 Excel 之后，不是结束。

真正下一步是点这个按钮。

## 12. 报价确认到底在确认什么

报价确认不是让你重新做人肉录入员。

它的目标是确认业务前提，而不是让你逐格核 Excel。

常见需要确认的问题包括：

1. 选择使用哪个产品代码
2. 货代是否确认你的产品可以承运
3. 是否包含巴西本地尾程派送
4. 是否包含清关服务
5. 是否包含进口税
6. 税费模式怎么选
7. 代缴税手续费是否适用
8. 报价是否仍在有效期内
9. 区域是按 CEP 自动匹配，还是先手动估算

## 13. 报价确认区怎么看

报价确认区通常会分成几块：

### 系统识别结果摘要

这里告诉你：

1. 识别到哪些产品代码
2. 识别到什么主报价
3. 识别到多少个尾程区域
4. 识别到多少条邮编区间
5. 是否识别到禁运或限制说明

### 业务选择

这里是你真正要做的选择：

1. 产品代码
2. 税费模式
3. 是否适用代缴税手续费
4. 尾程区域获取方式

### 业务确认问题

这里通常是“是 / 否 / 不确定”。

只有当必填问题不是“不确定”时，才更容易被系统确认通过。

### 高级识别详情

如果你不是物流同事，一般不用先看这个。

只有当你需要追查识别细节时，再展开查看。

## 14. 什么是“确认复杂度”

系统会给报价草稿一个复杂度等级。

### 可以确认

表示系统识别结构比较清楚，你只需要补少量业务确认。

### 还需要确认业务问题

表示系统已经识别到主要结构，但还需要你确认业务前提。

### 需要物流同事复核

表示系统识别到了一部分，但可能还缺关键结构，比如尾程矩阵或邮编表。

### 暂不可直接使用

表示当前报价结构还不够完整，建议不要直接用于正式测算。

## 15. 什么是 PRC 报价包

PRC 报价包通常不是只有一个“多少钱一公斤”。

它往往包含：

1. 产品代码
2. 主运费
3. 处理费重量段
4. 尾程派送矩阵
5. 邮编区域表
6. 限制品类说明

所以 PRC 报价更适合用“报价包”而不是“普通报价卡”来理解。

## 16. 如何输入 CEP 自动匹配区域

如果你知道巴西收件人 CEP：

1. 在单 SKU 测算中输入 CEP
2. 系统会优先使用当前货代报价文件自带的邮编区域表
3. 自动找到对应 zone
4. 再根据重量段和 zone 匹配尾程费用

系统会显示：

1. 匹配到的区域
2. 城市
3. 州
4. 当前使用的尾程费

## 17. 如果不知道 CEP 怎么办

可以先手动选择区域估算，例如：

1. SP-CAP
2. SP-INT1
3. SP-INT2
4. RJ-CAP
5. BA-INT4

这种方式适合报价前的快速估算。

## 18. 为什么不用全国固定 CEP 库

因为不同货代的尾程区域划分规则可能不同。

所以当前版本优先采用：

1. 货代报价文件自带的邮编区域表

这比全国统一 CEP 数据更接近实际计费逻辑。

## 19. 如何做单 SKU 测算

操作步骤：

1. 进入“单 SKU 测算”
2. 选择一张已确认报价卡
3. 输入 SKU 成本、尺寸、重量、售价
4. 如有需要，输入 CEP
5. 点击运行测算

结果会看到：

1. 利润总览
2. 报价卡使用情况
3. PRC 报价包与尾程匹配
4. 售价反推
5. 直邮拆解 / 本地仓拆解 / 渠道成本拆解
6. 风险提示

## 20. 如何做批量 SKU 测算

操作步骤：

1. 进入“批量 SKU 测算”
2. 上传 SKU CSV
3. 先看预览
4. 选择一张或多张报价卡
5. 点击运行批量测算
6. 查看结果表
7. 导出结果 CSV

## 21. 如何做多报价对比

操作步骤：

1. 进入“多报价对比”
2. 选择数据来源：
   - 当前单 SKU
   - 最近一次 CSV 预览
3. 勾选多张报价卡
4. 点击对比

它适合回答的问题是：

1. 哪个货代运费最低
2. 哪个货代利润最高
3. 哪个货代净利率最好
4. 哪个方案风险更低

注意：

不要只看最低运费，还要看：

1. 税费
2. 尾程费
3. 平台费
4. 总成本
5. 净利润
6. 报价状态

## 22. 常见风险提示怎么看

### 报价还没确认

说明这张报价卡还不适合直接拿去正式报价，只适合试算。

### 报价过期

说明报价有效期已过，建议重新向货代确认。

### 需要物流同事复核

说明系统虽然识别出了一部分，但还不能完全替代物流专业判断。

### 未输入 CEP

说明尾程区域可能仍然是手动估算。

### 合并单元格较多

说明原始 Excel 较复杂，系统已尽量处理，但建议关注业务确认结果。

## 23. 按钮点了之后，什么算成功

常见成功提示包括：

1. Excel 已读取
2. 已生成报价草稿
3. 已保存到报价库
4. 已打开报价确认区
5. 报价已确认并保存到报价库
6. 已匹配区域
7. 测算完成
8. 报价库已导出

如果按钮点了之后没有看到：

1. Toast 提示
2. 面板刷新
3. 自动滚动
4. 结果区域变化

就说明这一操作可能没有真正完成。

## 24. 如何导出给同事

### 导出报价库

适合共享报价基础数据。

步骤：

1. 进入“数据导入导出”
2. 点击“导出全部报价库 JSON”
3. 把 JSON 发给同事

### 导出批量测算结果

适合共享报价测算结果。

步骤：

1. 进入“批量 SKU 测算”
2. 运行批量测算
3. 点击“导出结果 CSV”

## 25. 同事如何导入继续使用

1. 打开工具
2. 进入“数据导入导出”
3. 导入报价库 JSON
4. 如果有需要，再去批量测算或多报价对比继续用

## 26. 当前数据保存在哪里

当前数据保存在本机浏览器 localStorage 中。

这意味着：

1. 更换浏览器后数据不会自动同步
2. 换电脑后数据不会自动同步
3. 清空浏览器缓存可能导致数据丢失

所以建议定期导出报价库 JSON 备份。

## 27. Dify 在这里到底是做什么的

Dify 不是主系统。

它只是一个可选增强工具，适合处理：

1. 特别复杂的 Excel
2. PDF 报价
3. 截图
4. 微信聊天里的半结构化报价

推荐流程是：

1. 先让本工具做 Excel 预处理
2. 如果本地规则识别还不够清楚，再复制 AI 可读文本给 Dify
3. 让 Dify 输出 `quote_cards.json`
4. 再导回本工具
5. 最终仍然由本工具完成确认和测算

## 28. 一句话记住这套工具的正确用法

先把报价整理成可复用的报价卡，确认后存进报价库，再反复用于 SKU 测算。

不要每次测算都重新解析原始报价。
