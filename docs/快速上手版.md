# 巴西 SKU 利润与物流报价决策器快速上手版

## 适合谁看

适合第一次拿到这套工具、希望 5 分钟内上手的同事。

## 这套工具一句话是干什么的

先把货代报价整理成可复用的报价卡，再用这些报价卡做单 SKU、批量 SKU 和多报价对比测算。

## 最推荐的使用顺序

1. 先去“报价卡管理”
2. 上传货代 Excel，或手动新增报价卡
3. 点击“保存草稿并进入报价确认”
4. 确认几个业务问题
5. 报价状态变成“已确认”
6. 去“单 SKU 测算”试一个产品
7. 再去“批量 SKU 测算”或“多报价对比”

## 第一次用，最简单的操作路径

### 路径 A：你手里有货代 Excel

1. 打开“报价卡管理”
2. 在“Excel 报价预处理”里上传 `.xlsx`
3. 等系统读取完成
4. 点击“保存草稿并进入报价确认”
5. 在“报价确认”里确认少量业务问题
6. 确认成功后，去“单 SKU 测算”选择这张报价卡

### 路径 B：你没有 Excel，但知道报价

1. 打开“报价卡管理”
2. 点击“新增报价卡”
3. 填写报价名称、货代、模式、计费方式和主要价格
4. 保存并确认
5. 去“单 SKU 测算”使用

## 上传 Excel 后，下一步到底做什么

上传成功后，不是结束。

正确下一步是：

1. 看系统是否识别到报价草稿
2. 点击“保存草稿并进入报价确认”
3. 在确认区确认业务问题

不要卡在“Excel 预处理结果”那里反复看表。

## 你需要确认的不是每个价格字段

系统会尽量自动识别：

1. 主报价
2. 处理费
3. 重量段
4. 尾程区域
5. 邮编区间
6. 产品代码

你主要确认这些业务问题：

1. 货代是否确认你的产品可以承运
2. 是否包含尾程
3. 是否包含清关
4. 是否包含进口税
5. 税费模式怎么选
6. 报价是否仍在有效期内

## 报价确认状态怎么理解

### 已确认

可以正式用于测算。

### 待确认

可以试算，但结果仅供参考。

### 已过期

价格可能已经失效，不建议直接正式使用。

### 已禁用

不再默认参与测算。

## 单 SKU 测算怎么做

1. 打开“单 SKU 测算”
2. 选择一张已确认报价卡
3. 填 SKU 成本、重量、尺寸、售价
4. 如果知道巴西 CEP，就填 CEP
5. 点击“运行单 SKU 测算”

重点看：

1. 净利润
2. 净利率
3. 风险提示
4. 当前使用的是哪张报价卡

## 批量 SKU 测算怎么做

1. 打开“批量 SKU 测算”
2. 上传 SKU CSV
3. 选择一张或多张报价卡
4. 点击“运行批量测算”
5. 导出结果 CSV

## 多报价对比怎么做

1. 打开“多报价对比”
2. 选择当前单 SKU，或最近一次批量 SKU
3. 勾选多张报价卡
4. 点击“对比多个报价”

不要只看最低运费，要一起看：

1. 总成本
2. 净利润
3. 净利率
4. 报价状态

## 如果知道巴西 CEP，建议一定要填

因为系统可以：

1. 自动匹配尾程区域
2. 自动估算更接近实际的尾程费用

如果不知道 CEP，也可以先手动选区域做估算。

## 常见提示是什么意思

### “已读取 Excel”

表示文件读进来了。

### “已生成报价草稿”

表示系统已经识别出可用草稿。

### “已保存到报价库”

表示这张报价可以反复复用。

### “已打开报价确认区”

表示下一步该做业务确认，不是回头再看 Excel。

### “测算完成”

表示结果已经更新。

## 怎么发给同事继续用

### 导出报价库

1. 去“数据导入导出”
2. 点击“导出全部报价库 JSON”
3. 把文件发给同事

### 同事导入

1. 打开工具
2. 去“数据导入导出”
3. 导入报价库 JSON

## 数据保存在哪里

当前数据保存在本机浏览器里。

这意味着：

1. 换电脑不会自动同步
2. 换浏览器不会自动同步
3. 清缓存可能会丢失

建议定期导出报价库 JSON 备份。

## Dify 在这里是什么角色

Dify 不是必须的。

只有当 Excel 太复杂、本地规则识别不够清楚时，才建议：

1. 复制 AI 可读文本
2. 发给 Dify
3. 让 Dify 输出 `quote_cards.json`
4. 再导回本工具

最终的确认和测算，仍然都在这套工具里完成。

## 最后记住这 3 句话

1. 先整理报价，再做测算。
2. 先确认报价卡，再正式定价。
3. 不要每次测算都重新解析原始报价。
